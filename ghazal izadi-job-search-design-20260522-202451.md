## Problem Statement

Ghazal is an Analytics/Data Engineer actively job hunting. She has a working Python
script (`job_tracker_updater.py`) that scrapes Indeed + LinkedIn via Apify, scores
listings, and outputs a styled Excel tracker. The gaps:

1. Only one correct resume (`cv_ghazal_izadi_AE.pdf`) — other variants have wrong
   name, email, phone number.
2. No company intelligence — can't distinguish fast-growing startups from slow enterprises.
3. No resume tailoring per job — currently manual.
4. No cover letter generation.
5. No learning loop — scoring weights are hardcoded, don't improve from outcomes.
6. Code is monolithic (617-line single script) with hardcoded config.
7. Resume file mapping is inconsistent with current local files (only canonical AE
   resume is confirmed present in `Resume/`).

She is NOT looking for volume (mass-apply). She wants 5 high-quality applications
per week at growth-stage, agile companies (50–500 headcount) where she can actually
have impact.

## What Makes This Cool

The system gets smarter over time. It starts with a canonical resume and generates
role-specific outputs dynamically. Company intelligence filters noise before tailoring.
Then a weekly feedback loop adjusts scoring using actual application outcomes.

Unlike mass-apply tools (ApplyPilot, LazyApply), this is a quality-targeting system.
It should help say NO to 95% of listings so the 5% that remain get real attention.

## Constraints

- Personal use (not a product). No auth, no multi-user, no SaaS complexity.
- Budget-conscious: Claude API usage should be minimal (only on approved jobs).
- Existing Apify setup (token + actors) must continue working.
- Excel output must stay — it's the review and tracking interface.
- Windows 11 environment (PowerShell + Python).
- Canonical resume is a PDF — must work from the PDF directly.

## Premises

1. Existing scraping logic is preserved but refactored into clean modules.
2. `cv_ghazal_izadi_AE.pdf` is the single source of truth for all resume variants.
   The AI generates role-specific variants (DE, DA, PA) dynamically from it.
3. Company intelligence (headcount + growth signals) is used as a hard pre-filter
   before tailoring runs. Tailoring only happens on jobs that pass the filter.
4. Quality over quantity: the design should make it easy to say NO to most jobs.
5. Architecture includes SQLite memory + feedback hooks from day one.

## Approaches Considered

### Approach A: Quick Win — Refactor + AI Bolt-On
Extend existing script with Claude API, keep Excel as source of truth.
Effort: S. Simpler but no learning, no clean architecture.

### Approach B (Recommended): Hybrid — deterministic core + constrained AI layer
Use Python for scraping/filtering/scoring/tracking, then use AI only for shortlisted
jobs to produce decision rationale and tailoring output.

## Recommended Approach: Hybrid

### New Capabilities

**1. Resume Agent (`src/agents/resume_agent.py`)**

Input: `cv_ghazal_izadi_AE.pdf`
Steps:
- Extract text via `pdfminer.six`
- Send to Claude API with prompt: "Extract skills, technologies, years of experience,
  and suggest 3-5 job titles this person should target."
- Cache extraction result in `output/resume_parsed.json` with PDF hash stored alongside:
  ```json
  { "pdf_md5": "abc123...", "extracted_text": "...", "skills": [...] }
  ```
  Re-parse if `hashlib.md5(open(pdf_path, 'rb').read()).hexdigest()` differs from stored hash.
  This catches resume updates automatically without needing to manually delete the cache.
- Generate role variant for a given title: Claude takes canonical text + target role
  and rewrites relevant sections (fixing name "Ghazal" not "Zahra", correct email/LinkedIn/phone)

Output: `output/resume_AE.txt`, `output/resume_DE.txt`, etc. (text variants for tailoring)

**2. Company Agent (`src/agents/company_agent.py`)**

Input: company name + URL
Sources (tried in order, first success wins):
1. `Apify LinkedIn Company Scraper` (primary) — employee range, description, industry
2. Claude API with company name as context (fallback) — infers size class from known companies
3. Unknown (both fail) — set stage="unknown", growth_score=5, do not reject

Output per company:
```json
{
  "name": "Acme Corp",
  "headcount_range": "51-200",
  "stage": "scale-up",
  "growth_score": 7,
  "source": "apify",   ← "apify" | "claude_inference" | "unknown"
  "verdict": "PASS"    ← or "REJECT: enterprise (10k+ employees)"
}
```

**Stage classification:**
- "startup": 1–50 employees
- "scale-up": 51–500 employees ← Ghazal's target zone
- "growth-stage": 501–2000 employees (acceptable)
- "enterprise": 2000+ ← REJECT unless exceptional match score

**Growth score signals (0–10, clamped):**
Data sources now:
- Recent Series A/B/C funding announcement: LinkedIn company description or news
  mention scraped via Apify (+3)
- Active hiring in data/engineering roles per LinkedIn job count (+2)
- Company founded in last 5 years (startup indicator) (+2)
- Layoffs mentioned in company description or name (−3)
- Default score for unknown companies: 5 (neutral, not rejected)

Future signals:
- LinkedIn follower growth year-over-year
- Glassdoor rating ≥ 4.0

**3. Company Filter (`src/filters/company_filter.py`)**

Jobs that FAIL the filter are marked "Filtered Out" in the tracker and not tailored.
Filter criteria (configurable in `config.yaml`):
```yaml
company_filter:
  reject_stages: ["enterprise"]
  min_growth_score: 4
  cache_ttl_days: 7
  reject_keywords_in_company_name: ["Consulting", "Staffing", "Recruiting"]

tailoring:
  min_score: 6  # only tailor jobs with match_score >= this
```

**4. Tailoring Agent (`src/agents/tailoring_agent.py`)**

Only runs on jobs that PASS the company filter AND have match_score >= 6.

Input: job description text (pasted manually — see `--tailor` workflow below) + canonical
resume text + target role
Output (text first; paste into your own final template):
1. `output/{company}_{date}/cover_letter.txt` — 3-paragraph cover letter referencing
   specific job requirements and Ghazal's matching experience
2. `output/{company}_{date}/resume_highlights.txt` — 3-5 bullet rewrites that echo
   the exact language in the job description (ATS optimization)

Claude prompt template:
```
SYSTEM:
You are a professional resume coach helping Ghazal Izadi, a senior Analytics Engineer
with 5+ years of experience in dbt, Airflow, Power BI, ClickHouse, and Python.
Her target roles are Analytics Engineer, Data Engineer, and Data Analyst.
Her canonical resume is provided. Always write in first person, professional tone.
Output strictly as JSON: {"cover_letter": "...", "highlights": ["...", "..."]}

USER:
Job description:
{job_description}

Resume text for this role (read from output/resume_{ROLE}.txt if it exists,
otherwise fallback to output/resume_parsed.json extracted text):
{resume_text}

Target role: {role_title} at {company_name}
```

Response must be parsed as JSON. If malformed (prose preamble, truncation):
```python
try:
    result = json.loads(response_text)
except json.JSONDecodeError:
    logger.error(f"Non-JSON response for {company_name}: {response_text[:200]}")
    return None  # caller does NOT write output or mark job "Tailored"
```

**4.1 LLM Shortlist Decision Layer (new priority)**

LLM review is constrained to shortlisted jobs only (for example, `match_score >= 7`),
not the full scraped set.

For each shortlisted job, produce:
- `Apply_Now` (yes/no)
- concise fit reason
- risk flags (missing requirements, seniority mismatch, ambiguity)
- 3 tailored resume bullet edits
- 1 cover-letter angle

This keeps costs low while improving decision quality where it matters.

**5. Excel Enhancements (`src/tracker/excel.py`)**

4 new columns appended to existing COLS:
- `Company Size` (headcount range string)
- `Company Stage` (startup/scale-up/growth-stage/enterprise)
- `Growth Score` (1–10 integer)
- `Intel Source` (apify | claude_inference | unknown — flags potentially inferred data)
- `Apply_Now` (yes/no from shortlist decision layer)
- `LLM_Reason` (short rationale for prioritization)
- `Risk_Flags` (optional, comma-separated)

Existing status workflow: New → Ready to Apply → Tailored → Applied → Interview → Rejected/Ignored

**6. Company Cache (`src/tracker/cache.py`)**

SQLite-backed cache for company intel. Prevents re-hitting the API for companies
already seen across runs.


### Workflow (end-to-end)

```
python main.py

1. Parse resume (Claude API) → output/resume_parsed.json  [skip if file exists]
2. Scrape Indeed + LinkedIn (Apify actors) → raw job list
3. For each unique company name — run concurrently (ThreadPoolExecutor, max_workers=3):
   a. Check CompanyCache (SQLite) — skip API if cached and < 7 days old
   b. If not cached: call Apify LinkedIn Company Scraper (primary data source)
      On failure: call Claude with company name + web context (fallback)
      On both failure: mark stage="unknown", growth_score=5, do not reject
   c. Write result to cache (SQLite handles concurrent writes safely)
   Typical runtime: 30 unique companies → ~60-90s with concurrency (vs 5-15min sequential)
4. Filter jobs:
   - HARD REJECT: stage="enterprise" or name contains Consulting/Staffing/Recruiting
     → Mark "Filtered Out" in tracker, do not tailor, show grayed out
   - SOFT WARN: growth_score < 4 (noisy signal; don't hide)
     → Keep in tracker, mark "Low Growth Signal" column, color yellow
     → Still tailor if match_score >= 6 (user decides)
5. Generate role-specific resume text variants for filtered-in jobs
   (Claude: canonical PDF text + target role → variant text)
6. Score remaining jobs (weighted scorer using config weights)
7. Save full list to Excel with 3 new company columns + Filtered Out status
   - Filtered jobs visible but grayed out
   - High-score passing jobs highlighted (score >= 7)
8. Send only shortlisted jobs (score threshold) to LLM decision layer
   - write `Apply_Now`, `LLM_Reason`, `Risk_Flags`
   - generate tailoring artifacts for apply-now jobs only

python main.py --tailor --jd job.txt

  (User saves job description to a .txt file first — reliable on all Windows terminals)

Prompts interactively:
  1. "Enter job URL to look up in tracker (or press Enter to skip):"
  2. Scans URL column in Excel (linear scan, fine for <500 rows)
     If URL not found or skipped: prompt "Company name? Role title?" and proceed without
     updating tracker status.
  3. Reads JD text from the --jd file path argument
  4. Calls tailoring_agent with JD text + role variant text (from `output/resume_{ROLE}.txt`
     for the matched role, or `output/resume_parsed.json` text if no variant exists) + role
  5. Saves cover_letter.txt + resume_highlights.txt to output/{company}_{date}/
  6. Marks job "Tailored" in Excel tracker
  7. Prints: "Ready → output/Acme Corp_20260522/"
```

### Error Handling

| Failure Point | Behavior |
|---|---|
| Apify scrape returns 0 results | Log warning, skip that category, continue |
| Company API fails (all fallbacks) | Set stage="unknown", growth_score=5, don't reject |
| Claude API fails on tailoring | Log error, don't write output, don't mark "Tailored" |
| Claude API fails on resume parse | Abort with clear message: "Set ANTHROPIC_API_KEY and retry" |
| Excel file locked (open in Excel) | Abort with: "Close job_tracker_ghazal.xlsx and retry" |

### Claude API Usage (cost estimate)

- Resume parsing: ~1000 tokens, once ever (cached in output/resume_parsed.json) = ~$0.003
- Role variant generation: ~1500 tokens × 4 roles, once per run if variants missing = ~$0.02
- Company classification (Claude fallback only): ~500 tokens per company = ~$0.001 each
- Tailoring per job: ~2000 tokens = ~$0.006
- Weekly run (50 new jobs, 5 pass filter, 3 tailored): ~$0.02 + 3×$0.006 = ~$0.04/week

Note: role variants (AE/DE/DA/PA) are generated ONCE per role and cached as .txt files,
not regenerated per job. Cost stays flat regardless of how many jobs are processed.

Total: under $2/month at typical job search intensity.

## Open Questions

1. **LinkedIn Company Scraper reliability for small companies:** The Apify
   `curious_coder~linkedin-company-scraper` actor may return incomplete data for
   startups with <50 employees or no LinkedIn presence. In practice, the companies
   Ghazal most wants to find (fast-growing Series A/B companies) tend to have solid
   LinkedIn profiles. Monitor hit rate in week 1; if it's <60%, consider the Apify
   "All Jobs Scraper" which includes company size as a field on the job listing itself.
2. **PDF generation for tailored resumes:** The cover letter + highlights are text
   output for now. Do you want the AI to produce a formatted PDF, or is a Word/text
   file sufficient for pasting into applications?
3. **Deduplication of companies:** If the same company appears in 5 job listings,
   company lookup runs once (cached). Cache TTL should be 7 days (companies don't
   change headcount weekly).
